# LANDrop

Простой инструмент для передачи файлов между компьютерами в локальной сети (LAN) с интеграцией в контекстное меню проводника Windows.

## Возможности

- 🔍 Автоматический поиск устройств в локальной сети через UDP-broadcast
- 📤 Отправка файлов по TCP с отображением прогресса (`tqdm`)
- 📥 Приём файлов в выбранную папку
- 🖱️ Интеграция с контекстным меню Windows: «Отправить через LANDrop» и «Принять файл сюда (LANDrop)»
- 📊 Прогресс-бар передачи как у отправителя, так и у получателя

## Как это работает

1. **Получатель** запускает `main_for_server.py`. Скрипт открывает TCP-порт `8080` и, пока никто не подключился, каждые 2 секунды рассылает по UDP широковещательное сообщение о себе (`scream()` из `Searcher.py`).
2. **Отправитель** запускает `main.py`. Скрипт слушает UDP-broadcast несколько секунд (`listener()`), собирает список доступных в сети устройств и предлагает выбрать нужное.
3. После выбора устройства отправитель подключается к получателю по TCP, отправляет JSON с метаданными файла (имя, размер), а затем сам файл чанками по 8192 байт.
4. Получатель сохраняет файл и показывает прогресс приёма.

## Требования

- Windows (используются `winreg` и `ctypes.windll` для контекстного меню и запроса прав администратора)
- Python 3.x
- Библиотека `tqdm`:
  ```bash
  pip install tqdm
  ```

## Установка

1. Скопируйте проект в постоянную папку — после установки меню пути к скриптам фиксируются в реестре.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Запустите `install_menu.py`. При необходимости скрипт сам запросит права администратора и добавит пункты в контекстное меню:
   ```bash
   python install_menu.py
   ```

## Использование

### Через контекстное меню (после установки)

- **Отправить файл:** ПКМ по файлу → «Отправить через LANDrop»
- **Принять файл:** ПКМ в пустом месте нужной папки → «Принять файл сюда (LANDrop)»

### Вручную, из консоли

Получатель:
```bash
python main_for_server.py "C:\путь\куда\сохранять"
```

Отправитель (без аргументов — выбор файла из папки `received_files`):
```bash
python main.py
```

Отправитель (с явным путём к файлу, как при вызове из контекстного меню):
```bash
python main.py "C:\путь\к\файлу.ext"
```

## Структура проекта

| Файл | Назначение |
|---|---|
| `main.py` | Клиент: поиск устройств, выбор файла/устройства, отправка файла |
| `main_for_server.py` | Сервер: приём подключения, приём файла, сохранение на диск |
| `Searcher.py` | UDP-логика: `scream()` — объявление о себе, `listener()` — поиск устройств |
| `install_menu.py` | Установка пунктов в контекстное меню Windows через реестр |
| `check_sum.py` | 🚧 Заготовка под проверку контрольной суммы (пока пусто) |

## Известные ограничения

- Проверка целостности файла (checksum) пока не реализована.
- Обработка сетевых ошибок (обрыв соединения, повторная отправка) минимальна.
- Работает только на Windows из-за использования `winreg` и `ctypes.windll`.
